## Workflow

### Phase 1: Requirements & Users
- Clarify the user goal and context (desktop/mobile)
- Identify key workflows and pain points

### Phase 2: IA + Flow
- Define navigation and screen structure
- Define primary/secondary actions and states

### Phase 3: Visual System
- Apply consistent spacing/typography
- Define components and variants (buttons, inputs, tables, cards)

### Phase 4: Accessibility & QA
- WCAG checks (contrast, focus, labels)
- Responsive checks (small screens, touch targets)
